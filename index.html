Analyzer — результаты

Каждый присланный скриншот прогнан через n8n-воркфлоу. Слева — исходник, справа — результат. По умолчанию открыт сырой вывод воркфлоу в формате ТЗ; тумблер «Разбор» показывает тот же результат в наглядном виде.

Как реализовано определение сторон и платформы +

Стороны переписки

Автор каждого сообщения определяется только по горизонтальному выравниванию пузыря: прижат к правому краю → владелец переписки (ПРАВО), к левому → собеседник (ЛЕВО). Цвет пузыря не используется намеренно — при кастомных темах он произвольный, а выравнивание остаётся инвариантом. Модель возвращает сторону как left/right через structured output (function-calling), а не свободным текстом.

Платформа

Распознаётся по UI-сигнатурам (шапка + строка ввода), а не по умолчанию: Telegram (плоские пузыри, галочки, скрепка), WhatsApp (хвостики, двойные галочки, дудл-фон), Instagram Direct (ник вместо имени, камера в строке ввода, градиентные пузыри), Tinder, Twinby («% совпадения», дейтинг). Поле platform_type делит на мессенджер / сайт знакомств. Недостаточно улик → unknown, без угадывания.
